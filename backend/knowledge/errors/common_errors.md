# 计量经济学常见错误及解决方案

本文档收集了实证分析中常见的错误模式及正确做法。

---

## 1. 因果识别错误

### 错误 1.1：混淆相关与因果

**错误示例**：
> "回归结果显示，教育年限每增加一年，收入增加 8%，因此教育导致收入增加。"

**问题**：OLS 回归只能说明相关关系，不能说明因果关系。可能存在：
- 能力偏误：能力高的人既接受更多教育也挣更多钱
- 家庭背景：富裕家庭的孩子既能接受更多教育也有更多机会

**正确做法**：
1. 使用工具变量（如强制入学法律作为教育的 IV）
2. 利用自然实验
3. 如果没有因果识别策略，只描述相关关系

**正确表述**：
> "回归结果显示，教育年限与收入正相关，教育年限每增加一年，收入平均高 8%。但这一结果可能受到遗漏变量偏误影响。"

---

### 错误 1.2：DID 未验证平行趋势

**错误示例**：
直接报告 DID 结果，没有展示平行趋势检验。

**问题**：DID 的有效性完全依赖于平行趋势假设。

**正确做法**：
```python
# 1. 事件研究图
# 2. 政策前系数联合检验
# 3. 如果平行趋势有疑问，使用替代方法
```

---

### 错误 1.3：IV 第一阶段太弱

**错误示例**：
> "F 统计量为 5.2，因此工具变量有效。"

**问题**：F < 10 表示弱工具变量，2SLS 估计量会向 OLS 偏倚。

**正确做法**：
1. F > 10 是基本门槛
2. 如果 F 较低，使用 LIML 估计量
3. 报告 Anderson-Rubin 置信区间
4. 寻找更强的工具变量

## 2. 标准误计算错误

### 错误 2.1：聚类层级不当

**错误示例**：
使用省份政策研究，但按个人层面计算标准误。

**问题**：
- 同一省份的个体误差相关
- 忽略组内相关会严重低估标准误
- 导致虚假显著

**正确做法**：
```python
# 聚类到政策变异的层级
model.fit(cov_type='cluster', cov_kwds={'groups': df['province']})
```

**聚类层级选择原则**：
- 聚类到处理变异的层级
- 如果处理在省份层面变化，聚类到省份
- 如果处理在个人层面变化，可以使用个人聚类或稳健标准误

---

### 错误 2.2：聚类数量太少

**错误示例**：
> "数据包含 10 个省份，按省份聚类。"

**问题**：聚类数量少于 30-50 时，聚类标准误可能不准确。

**正确做法**：
1. 使用 Wild Cluster Bootstrap
2. 报告不同聚类方式的结果
3. 讨论小聚类数的局限性

```python
# Wild Cluster Bootstrap（需要特殊包或手动实现）
from wildboottest import boottest
boottest(model, cluster=df['province'], bootstrap='wild')
```

## 3. 样本选择错误

### 错误 3.1：选择性样本偏误

**错误示例**：
研究"MBA 对收入的影响"，只使用获得 MBA 的人的样本。

**问题**：需要有对照组才能识别因果效应。

**正确做法**：
1. 包含有可比性的对照组
2. 使用匹配方法平衡特征
3. 讨论选择性问题

---

### 错误 3.2：幸存者偏误

**错误示例**：
研究"创新对企业绩效的影响"，只使用存续企业数据。

**问题**：创新失败的企业可能已经退出市场，只看存续企业会高估创新效果。

**正确做法**：
1. 使用包含退出企业的数据
2. 建模选择过程（Heckman 选择模型）
3. 明确说明样本限制

---

## 4. 变量测量错误

### 错误 4.1：因变量和自变量使用同源数据

**错误示例**：
用企业自评的"管理水平"解释企业自评的"创新能力"。

**问题**：共同方法偏差（Common Method Variance）会人为夸大相关性。

**正确做法**：
1. 使用不同来源的数据
2. 使用客观测量指标
3. 报告共同方法偏差检验结果

---

### 错误 4.2：使用比率变量回归

**错误示例**：
$$\frac{Y}{X_1} = \alpha + \beta X_2 + \epsilon$$

**问题**：
- 分子分母相关导致虚假关系
- 常见于财务比率分析

**正确做法**：
```python
# 方法 1：水平值回归，控制规模
Y ~ X2 + log(X1)

# 方法 2：对数变换
log(Y) ~ X2 + log(X1)
```

## 5. 结果解读错误

### 错误 5.1：混淆统计显著与经济重要

**错误示例**：
> "政策效应显著（β = 0.001, p < 0.01），因此政策非常有效。"

**问题**：大样本下即使很小的效应也会统计显著，但可能没有实际意义。

**正确做法**：
1. 报告效应大小（百分比变化、标准化系数）
2. 讨论经济意义
3. 与其他研究对比

**正确表述**：
> "政策使收入增加 0.1%，虽然统计显著，但经济意义有限。这一效应远小于某某研究发现的 5% 效应。"

---

### 错误 5.2：对数变量系数解读错误

**错误示例**：
> "ln(Y) 对 X 的系数为 0.15，说明 X 增加 1 单位，Y 增加 15%。"

**问题**：只有当系数较小（< 0.1）时，才能近似解读为百分比。

**正确做法**：
```python
# 精确计算百分比变化
pct_change = (np.exp(0.15) - 1) * 100  # = 16.2%
```

**各种情况的解读**：
- ln(Y) ~ X：X 变化 1 单位，Y 变化 100β%（β 小时）
- Y ~ ln(X)：X 变化 1%，Y 变化 β/100 单位
- ln(Y) ~ ln(X)：弹性，X 变化 1%，Y 变化 β%

---

### 错误 5.3：忽视多重假设检验

**错误示例**：
对 20 个子样本分别回归，报告 3 个显著的结果。

**问题**：如果做 20 次检验，即使没有真实效应，也有约 1 个会在 5% 水平上显著。

**正确做法**：
1. 报告所有检验结果
2. 使用多重检验校正（Bonferroni、FDR）
3. 区分探索性和验证性分析

```python
# Bonferroni 校正
n_tests = 20
adjusted_alpha = 0.05 / n_tests  # = 0.0025
```

## 6. 数据处理错误

### 错误 6.1：不当的缺失值处理

**错误示例**：
直接删除所有含缺失值的观测，样本量从 10000 减少到 2000。

**问题**：
- 可能导致样本选择偏误
- 损失大量信息

**正确做法**：
1. 分析缺失模式
2. 如果 MAR，考虑多重插补
3. 报告缺失值处理方式及影响

---

### 错误 6.2：过度 Winsorize

**错误示例**：
对所有变量在 1% 和 99% 分位 Winsorize。

**问题**：
- 可能删除真实的有信息量的观测
- 人为压缩分布

**正确做法**：
1. 只对确实有异常值问题的变量处理
2. 检验结果对 Winsorize 的敏感性
3. 报告原始数据和处理后数据的结果

---

## 7. 汇报错误

### 错误 7.1：选择性汇报

**错误示例**：
只报告显著的模型设定，不报告其他设定的结果。

**问题**：P-hacking，会导致发表偏倚。

**正确做法**：
1. 预注册分析计划
2. 报告主要分析和所有预设的稳健性检验
3. 使用 specification curve 展示结果稳健性

---

### 错误 7.2：不报告关键诊断

**错误示例**：
IV 分析不报告第一阶段 F 统计量。

**正确做法**：

| 方法 | 必须报告 |
|------|----------|
| DID | 平行趋势检验图、事件研究系数 |
| IV | 第一阶段 F、过度识别检验（如适用） |
| RDD | 断点操纵检验、带宽敏感性 |
| PSM | 匹配后平衡性检验 |

## 快速错误检测脚本

```python
def check_common_errors(model_results, df, treatment_var=None, cluster_var=None):
    """检测常见计量错误"""
    warnings = []

    # 1. 检查样本量
    if len(df) < 100:
        warnings.append("⚠️ 样本量较小 (<100)，结果可能不稳定")

    # 2. 检查聚类
    if cluster_var:
        n_clusters = df[cluster_var].nunique()
        if n_clusters < 30:
            warnings.append(f"⚠️ 聚类数量较少 ({n_clusters})，考虑 Wild Bootstrap")

    # 3. 检查多重共线性
    # (VIF > 10 警告)

    # 4. 检查效应大小
    for var, coef in model_results.params.items():
        if abs(coef) < 0.01 and model_results.pvalues[var] < 0.05:
            warnings.append(f"⚠️ {var} 统计显著但效应很小，检查经济意义")

    # 5. 检查 R²
    if hasattr(model_results, 'rsquared'):
        if model_results.rsquared > 0.99:
            warnings.append("⚠️ R² 过高，检查模型设定是否有问题")
        if model_results.rsquared < 0.01:
            warnings.append("⚠️ R² 很低，模型解释力有限")

    return warnings
```
